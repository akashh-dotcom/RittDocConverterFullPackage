### 1. Copyright Metadata Concatenation — Year + Author Name

- **Severity:** HIGH
- **Source:** Converter
- **Scope:** Systemic (all 17 chapter copyright blocks)
- **Known Issue ID:** C-002
- **Description:** The copyright year "2025" is concatenated directly with the author initial "A." producing `2025A. Zbar` instead of the year being on a separate line from the author name. All copyright metadata fields (copyright statement, author, book title, DOI, full author name, affiliation) are merged into a single `<para>` without `<?lb?>` processing instructions to create line breaks.
- **Root Cause:** The converter's copyright handler uses `extract_text()` which flattens all child elements into a single string. Separate `<p>` and `<span>` elements in the source HTML that represent distinct metadata fields get concatenated without separators. No `<?lb?>` processing instructions are inserted between fields.
- **Example:**

- **Source:** Converter
- **Scope:** Systemic (14 instances across 8 reference files)
- **Known Issue ID:** C-006
- **Description:** Double periods (`..`) appear in bibliography entries where a citation's text ends with a period and the DOI URL follows, producing patterns like `225..` or `2874..` before the DOI link.
